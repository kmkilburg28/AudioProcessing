Vector 'x'
[0, 0.7071, 1, 0.7071, 0, -0.7071, -1, -0.7071]

Note: you may want to use sqrt(2)/2 instead of 0.7071.

Result
[0 + 0i, 0 + -4i, 0 + 0i, 0 + 0i, 0 + 0i, -0 + 0i, 0 + 0i, -0 + 4i]

Compare your output with the output generated by MATLAB's fft() method for the same vector 'x'. Include the result below, and point out any discrepancies. You may also use one of the FFT libraries above, if you choose.

Matlab Result & Analysis
[0.0000 + 0.0000i   0.0000 - 4.0000i   0.0000 + 0.0000i   0.0000 + 0.0000i   0.0000 + 0.0000i   0.0000 - 0.0000i   0.0000 + 0.0000i   0.0000 + 4.0000i]

There are two primary discrepancies.
The first is that my FFT produces complex numbers where some of the real constants equal to 0 are positive and others are negative; however, the MATLAB implementation produces purely positive signed zeros.
The second is that my FFT is has higher precision (less rounding) the MATLAB implementation. While MATLAB shows 4 decimal digits of precision and mine shows 0, this is simply a display discrepency. If I use the values of 0.7071 in my FFT, then my resulting vector shows decimal points as the constants are not integers. This is due to 0.7071 being a 4 decimal digit rounding of sqrt(2)/2. However, MATLAB does not change for either input because it has a lower rounding threshold.
